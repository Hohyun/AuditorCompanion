[ QUERY SYNTAX ]

[Keyword matching]

foo→ 기본 필드에 대해 키워드 foo 검색 (현재 기본필드는 contents로 지정해 놓았음)
"foo bar"→ 기본 필드에 대해 phrase "foo bar" 검색

[Fileds Operator]

file-name:foo→"file-name" 필드에 대해 foo 검색
현재 Index 내의 필드 명 → path, file-name, full-file-name, contents, file-size, md5, author, application-name, creation-date, last-midified, last-author, page-count

[Boolean Operator]

OR → 기본값
AND, +→ 반드시 있어야 함
NOT, -→ 해당 단어 제외
※ 반드시 대문자이어야 함
foo OR bar | foo bar→ 기본값이 or 임, 이 중 하나만 있으면 됨
foo AND bar | +foo +bar→ foo와 bar가 둘 다 있어야 함
foo -bar→ foo는 있고 bar는 없어야 함
+foo bar → foo는 반드시 있고 bar는 있어도 없어도 됨

[Grouping]

(foo OR bar) AND fox

[Wildcard Searches]

foo*→ 기본 필드에 대해 foo로 시작되는 모든 단어
foo*bar→ 기본 필드에 대해 foo로 시작해서 bar로 끝나는 단어 검색

[Proximity Searches]

"foo bar"~4→ foo와 bar가 서로 4 단어 이내에 존재

[Range Searches]

page-count:[1 to 5]→ page 수가 1에서 5 사이인 문서

[Boosting a Term]

file-name:foo^4 bar→ file-name에 가중치 4를 줌

[Fuzzy Searches]

roam~→ foam, roams 와 같이 비슷한 단어를 찾아 줌

[Escaping Special Characters]

+ - && || ! ( ) { } [ ] ^ " ~ * ? : \ → 이 글자를 지정하려면 앞에 \를 붙인다 (\+, \- ...)
